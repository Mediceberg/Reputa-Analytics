أنت UI/UX Engineer و Frontend Architect مختص في تطبيقات Pi Network و Pi Browser.
مهمتك إصلاح وتوحيد تجربة التطبيق الحالية ليعمل بشكل متناسق
سواء كـ Web App عادي أو كتطبيق داخل Pi Browser (Mobile App-like).

السياق:
- يوجد اختلاف واضح بين الواجهة عند فتح التطبيق:
  • كموقع ويب
  • كـ App داخل Pi Browser
- توجد أيقونات وصفحات ناقصة أو غير ظاهرة
- توجد تعارضات في التصفح، الدخول، الخروج، والتنقل
- الواجهة الحالية غير محسّنة كـ Mobile App

المطلوب بالتحديد:

1) توحيد تجربة التصفح (Navigation):
- إصلاح مشاكل:
  • الدخول (Login)
  • الخروج (Logout)
  • التنقل بين الصفحات
- ضمان أن التصفح يعمل بسلاسة:
  • داخل Pi Browser
  • داخل المتصفح العادي
- منع إعادة التحميل غير الضرورية
- الحفاظ على state بين الصفحات

2) معالجة الاختلاف بين Web و App:
- التأكد أن جميع الأيقونات والصفحات متوفرة في الوضعين:
  • Wallet Info
  • Audit Report
  • Activity
  • Profile
- عدم إخفاء أي صفحة في وضع Pi Browser
- توحيد Layout والمنطق مع تكييف العرض فقط (Responsive)

3) إضافة قائمة جانبية (Hamburger Menu) في وضع App:
- عند الضغط على أيقونة ☰ (ثلاثة خطوط):
  • تظهر قائمة كاملة
  • تضم جميع الصفحات
- تحسين تجربة الاستخدام بيد واحدة
- القائمة تظهر فقط في وضع Mobile / Pi Browser

4) تحسين عرض رصيد المحفظة:
- إظهار كمية Pi التي تمتلكها المحفظة:
  • في الواجهة الرئيسية
  • في صفحة Profile
- البيانات يجب أن تكون:
  • حقيقية عند الربط بـ Pi
  • Demo data في وضع Demo

5) تحسين واجهة Profile:
- تبسيط التصميم
- إزالة التعقيد البصري
- عرض معلومات أساسية فقط:
  • Username
  • Wallet Address
  • Balance
  • Reputation Score
  • Level
- تنظيم المحتوى بشكل واضح وقابل للقراءة

6) تحسين واجهة Daily Check-in:
- جعلها:
  • واضحة
  • سهلة القراءة
  • غير معقدة
- عرض:
  • عدد الأيام
  • النقاط المكتسبة
- قسم نقاط الإعلانات:
  • يبقى مغلق (Disabled)
  • مع توضيح أنه غير مفعل حاليًا

7) إصلاح واجهة Recent Transactions:
- معالجة تعارض النصوص والأرقام في وضع الهاتف
- تحسين:
  • حجم الخط
  • التفاف النص
  • المسافات
- ضمان أن الواجهة تعمل بشكل سليم داخل Pi Browser

8) تحسين شعار التطبيق:
- تكبير الشعار في منتصف الواجهة قليلًا
- إزالة الخلفية السوداء
- جعله أوضح وأكثر احترافية كتطبيق

قيود مهمة:
- لا حذف أي Feature موجود
- لا تغيير منطق البروتوكول
- فقط إصلاح + تحسين + توحيد تجربة المستخدم
- الحفاظ على الأداء وخفة التطبيق داخل Pi Browser

النتيجة المطلوبة:
- تطبيق متناسق
- يعمل كـ App حقيقي داخل Pi Browser
- نفس الميزات بين Web و App
- تجربة استخدام سلسة، واضحة، ومناسبة للهاتف
