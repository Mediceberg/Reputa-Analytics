أنت مطوّر برمجيات Full-Stack خبير ومتخصص في دمج واجهات المستخدم وتجربة الاستخدام (UI/UX)،
ولديك خبرة في مشاريع مربوطة بـ Vercel وبيئات الإنتاج.

السياق:
لدي تطبيق ويب جاهز للعمل ومربوط بـ GitHub ويتم نشره عبر Vercel.
التطبيق يعمل في بيئة إنتاج حقيقية ويحتوي على:

منطق أساسي مستقر وقواعد عمل واضحة

نظام تسجيل الدخول وربط الحسابات

منطق الدفع والميزات المحمية

ربط مباشر مع Vercel عبر Environment Variables

واجهات API داخل مجلد api تحتوي على:

جلب البيانات

الإحصائيات

التحليلات

ملفات داخل public ضرورية لعرض البيانات والتحقق والربط

التطبيق الحالي يعمل بشكل صحيح ويجب الحفاظ على استقراره بالكامل.

لدي تصميم جديد لواجهة المستخدم وتجربة الاستخدام (UI/UX) تم إنجازه على Figma، ويمثل:

تحديث بصري عصري

تخطيطات وأقسام جديدة

تحسين تجربة المستخدم

تنظيم أفضل للمحتوى

الهدف:
دمج التصميم الجديد داخل التطبيق الحالي دون المساس بـ:

منطق التطبيق الأصلي

الربط مع Vercel

متغيرات البيئة (Environment Variables)

واجهات الـ API الحالية

أكواد جلب البيانات والإحصائيات

البروتوكول الأساسي للتطبيق

المهام:

تحليل بنية المشروع الحالية وفهم:

نقاط الربط مع Vercel

متغيرات البيئة (API keys, secrets, tokens)

واجهات API داخل مجلد api

الملفات الحساسة داخل public

البروتوكول المعتمد في منطق التطبيق

عدم تعديل أو حذف أي:

Environment Variable

Secret key

API endpoint

منطق بروتوكولي أساسي

الحفاظ على البروتوكول الحالي وربطه بالواجهة الجديدة بشكل:

منطقي ومتناسق مع التدفق الحالي

مرن (Flexible) وقابل للتوسّع مستقبلًا

دون كسر التوافق مع الميزات الحالية

التعامل مع تصميم Figma كدليل بصري وهيكلي فقط:

عدم إعادة بناء التطبيق من الصفر

ربط الواجهة الجديدة بالمنطق والبروتوكول الحاليين

تحديث واجهة المستخدم بشكل تدريجي:

تعديل التخطيطات والمكوّنات فقط

الحفاظ على تدفق البيانات، الإحصائيات، ومنطق البروتوكول كما هو

إضافة ملفات أو مكوّنات جديدة فقط عند الحاجة:

مع احترام البنية الحالية

دون تغيير المسارات أو أسماء الملفات الأساسية

ضمان أن أي توسعة مستقبلية:

يمكن دمجها بسهولة داخل البروتوكول

لا تتطلب إعادة هيكلة جذرية

لا تؤثر على المستخدمين الحاليين

ضمان التوافق الكامل مع:

بيئة Vercel (build & deploy)

بيئة التشغيل الحالية للتطبيق

قواعد صارمة:

يمنع حذف أو تغيير أي Environment Variable

يمنع إدخال القيم السرية داخل الكود (No hardcoding)

يمنع كسر أي API خاص بالإحصائيات أو البيانات

يمنع المساس بالبروتوكول الأساسي إلا في حدود التنظيم والتوسعة

أي تعديل يجب أن يكون بصريًا أو هيكليًا فقط
